1. Real time censoring

This project is solely work on my own and the findings, how it possibly works or the way it works are entirely determined by me.

SCOPE : The real scope of this project is to implement censor based system on streamming networks to convert the explicit contents on tv or any movies into family oriented content, Which could really come in handy when you are watching with family.

WORKING : Working is made simple as the sensor's function is governed by three constraints.

  • When there is a explicit scene such as kiss scene or nude scene or cuss words, it gets trimed as seemless.
  • If the explicit scene is pivotal to the movie or serves as a turning point in the story, it undergoes automatic CGI enhancement to seamlessly integrate it.
  • On the note of using the computer graphics it is basically reverse engineering the Deepfake algorithm.

How it's done is by the implementation of computer vision and NLP algorithms to learn more about it read my reasearch work on this here.

2. Emotion tweet detector

I wouldn't say this project is solely developed by me. As part of my learning journey in the field of Artificial Intelligence, this project is an exciting endeavor focusing on emotion detection.

SCOPE : The scope is basically simple when we provide a particular tweet to the ML model, it ranges as in rate the tweet's emotional condition as positive, negative or neutral.

WORKING : The process resembles Google's Teachable Machine. We provide the necessary tweet data to train the model, enabling it to recognize emotions in new tweets based on the patterns it learned from the training data.

Personal Virtual Private Network

As you may know VPN works by establishing a digital connection between your computer and a remote server owned by a VPN provider, Here we are not relaying on any third party VPN provider as we are creating a remote server on our own by cloud computing services.

WORKING : For starters we are cretaing an Virtual machine on the Cloud server, it is called EC2 Instance in AWS. Through we generate a key pair for SSH access.

Now for the big part we are using the Opensource OpenVPN which creates a secure tunnel for data traffic to pass between the VPN client and server. Which makes its accessible so easy.

I've documented the entire process in a blog post which can be found in my blog.

Smart Watch Open source OS - Project Kaizen

This project has been on my mind for a long time, and I'm still experimenting with various aspects of it, ranging from kernel development to firmware connections to building an operating system.

As everyone knows, Apple, Google, and Samsung dominate the smartwatch industry. One of the key factors contributing to their success in the wearables market is the wide variety of options they offer in their watch operating systems. These OSs are optimized to deliver fast and accurate results on relatively limited hardware, particularly in terms of health-related functionalities. Which ranges from offering all health related options on a small system which is more optimized to work as fast and give accurated results on less likely hardware.

The answer depends on the perfection of stuff they are doing is mainly because of the R&D which makes all of this possible. But what they are doing is mostly closed source work as we never know the architecture on which the OS is built upon. This project which is an open source of building the OS with the ground architecture from building a customizable kernel to designing the user interface to deploying the apps to it.
